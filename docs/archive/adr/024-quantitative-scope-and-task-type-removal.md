# ADR-024: hitl-metrics の責務範囲 — 定量指標と定性評価の分離

## ステータス

採用済み

## 関連 ADR

- 依存: ADR-018（メトリクス体系の前提を引き継ぎつつ、task_type 集計分類を部分廃止）
- 関連: ADR-022（ダッシュボードのアクショナビリティ改善）
- 関連: ADR-023（PR 単位のトークン消費効率メトリクス）

## コンテキスト

ADR-018 で task_type（branch プレフィックスから抽出）を導入し、ADR-022 で dashboard に「タスク種別 avg tokens / PR」panel を追加した。
ADR-023 では PR 単位効率指標 `tokens_per_tool_use` / `pr_per_million_tokens` を導入したが、PR の複雑度がコントロールされていないため、これらを状態評価軸として使うと誤誘導になることが今回の dashboard 再編で判明した（PR 単位指標は外れ値検出の用途に限定する方針へ）。

これに付随して、task_type についても以下の構造的な脆さが顕在化した：

1. **branch 命名規約への依存**: `feat/`, `fix/`, `chore/`, `docs/` のプレフィックスを正しく付ける運用前提。`feature/...` のような揺れで分類が崩れる
2. **命名と内容のズレ**: feat ブランチに実は fix だけが入る、chore ブランチに本格機能が含まれるなど、branch 名と PR 実態の乖離は日常的に発生する
3. **集計前提の崩壊**: task_type ごとの平均比較は「同種タスクは複雑度が近い」を暗黙前提とするが、この前提自体がデータから検証できない

より根本的に、PR の内容（意味・品質・適切さ）を計測しようとすれば、コードレビューと同等の意味解析が必要になる。これはルールベース／集計ベースのツールでは原理的に表現できず、LLM のような定性評価器に委ねるべき領域である。

## 決定

### 案 A: 定量指標に責務を限定し、task_type を集計軸から廃止する（採用）

hitl-metrics の責務範囲を以下に明確化する：

- **担うこと**:
  - 定量指標による**状態評価**（活動量・コスト・スループット・期間集計効率・質の推移）
  - **問題特定の入口**としての外れ値検出（PR 単位）
- **担わないこと**:
  - PR 内容の意味的評価
  - 指示品質の妥当性判断
  - コードレビューの内容評価
  - これらは LLM 評価層に委ねる

実装上の変更（既に feat/dashboard ブランチで実施済み）：

- 「タスク種別 avg tokens / PR」panel を削除
- PR スコアカードから task_type 列を削除
- リワークホットスポット panel を削除（PR スコアカードのソート機能で代替可能、`changes_requested` の色分け強調は PR スコアカードに統合）
- `sessions.task_type` カラム自体は schema に残す（既存データの後方互換性、将来の任意フィルタとしての利用余地）

### 案 B: task_type 分類を維持しつつ精度を上げる（却下）

GitHub PR label の活用、PR title 解析、commit log からのカテゴリ推定など、より堅牢な分類ソースに切り替える案。

却下理由：

- どの推定方式も表層的なシグナルに依存し、原理的脆さは解消しない（label の付け忘れ、title の揺れ等）
- 推定精度を上げるコストに対し、得られる行動可能性が限定的（同種タスクの平均比較自体に意味があるか怪しい）
- 「定量指標で扱える限界を超えた」と認めて LLM に委ねる方針と整合しない
